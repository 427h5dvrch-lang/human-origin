# HumanOrigin Cartouche Standard

## Purpose

The HumanOrigin cartouche is the visible documentary proof mark attached to a document.
It is a visible trust layer.
The authoritative proof remains the portable proof file.

## Product principle

HumanOrigin should feel less like a brand mark and more like a discreet international documentary standard of verifiable provenance.

It must be:
- sober
- stable
- readable
- internationally credible
- non-promotional

It must not feel like:
- a startup badge
- a marketing label
- a decorative certificate
- a web3 or NFT mark
- a heavy administrative stamp

## Official language

Preferred visible language:
- `HumanOrigin`
- `Document-bound proof`
- `ID: ...`
- `Verify record`
- `Doc: ...` (optional)

Avoid as primary visible language:
- `Certified by HumanOrigin`
- `Officially verified`
- `Trusted human origin`
- `Authentic human work`
- `Premium verified`

## Official hierarchy

1. `HumanOrigin`
2. `Document-bound proof`
3. `ID: ...`
4. `Verify record`
5. `Doc: ...` (optional)
6. QR code

## Official variants

### Compact standard
```text
HumanOrigin
Document-bound proof
ID: HO-27A4-91C2
Verify record
Doc: FE61-9168-…-9489
[QR]
```

### Ultra-compact standard
```text
HumanOrigin
Document-bound proof
ID: HO-27A4-91C2
Verify
[QR]
```

## Visual direction

Target mix:
- 60 percent documentary seal
- 30 percent academic standard
- 10 percent premium editorial finish

Recommended composition:
- horizontal rectangle
- text block on the left
- QR proof cell on the right
- clear vertical separation
- minimal accents

## Placement standard

Default placement:
- **bottom-right of the first page**

By document type:
- PDF report / manuscript / dossier: bottom-right of first page
- Final published multipage PDF: bottom-right of each page, compact repeated version
- Cover page: top-right possible if useful, but not default
- Image: bottom-right, compact variant

## Standardization principle

HumanOrigin becomes recognizable only if wording, hierarchy, placement, and silhouette remain stable.

## System hierarchy

1. Portable proof file
2. Document binding
3. Verifier
4. Visible cartouche
5. OPEN_FIRST and package explanation files
