# HumanOrigin Cartouche Standard

## Purpose

The HumanOrigin cartouche is the visible documentary proof mark attached to a document.

Its role is not to replace the proof file.
Its role is to:

1. signal that a verification path exists,
2. establish a visible documentary status,
3. point toward the reference proof,
4. make HumanOrigin recognizable as a standard.

The cartouche is a visible trust layer.
The authoritative proof remains the portable proof file.

---

## Product principle

HumanOrigin should feel less like a brand mark and more like a discreet international documentary standard of verifiable provenance.

The cartouche must therefore be:

- sober,
- stable,
- readable,
- document-compatible,
- internationally credible,
- non-promotional.

It must not feel like:
- a startup badge,
- a marketing label,
- a decorative certificate,
- a web3 / NFT mark,
- a heavy administrative stamp.

---

## Official language

### Core language
The official documentary language is:

- `HumanOrigin`
- `Document-bound proof`
- `ID: ...`
- `Verify record`
- `Doc: ...` (optional visible short form)

### Why this language
This language is preferred because it is:
- rigorous,
- portable,
- non-promotional,
- compatible with incomplete and certified states,
- durable as a standard.

### Language to avoid
Avoid as primary visible language:
- `Certified by HumanOrigin`
- `Officially verified`
- `Trusted human origin`
- `Authentic human work`
- `Premium verified`
- any wording that overclaims or sounds promotional.

---

## Official cartouche hierarchy

### Standard hierarchy
1. `HumanOrigin`
2. `Document-bound proof`
3. `ID: ...`
4. `Verify record`
5. `Doc: ...` (optional)
6. QR code

### Meaning of each line
- `HumanOrigin` = system identity
- `Document-bound proof` = documentary status
- `ID` = short reference identifier
- `Verify record` = action
- `Doc` = visible short document reference
- `QR` = access to verification path
